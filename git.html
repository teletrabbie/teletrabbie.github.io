Wichtiges beim Arbeiten mit Git

Typischer Ablauf, wenn online Repo existiert

  1. git clone [url]: zum Klonen eines online Repos
  2. Committen
    • Entweder
      1. git add [filename]: nur diese eine Datei wird beim nächsten Commit gespeichert
      2. git commit -m "message": das ist quasi das Gleiche wie das "Speichern" einer Datei
    • oder
      • git commit -am "message": alle veränderten Dateien werden committet
  3. git push: damit werden die Änderungen tatsächlich erst online aktiv

Wichtige weitere Befehle

  • git status (nach dem commit steht etwas von "1 commit ahead" oder so)
  • git pull
    • Gegenteil von git push, d.h. die Online-Version auf meinen Rechner heruntergeladen
    • Sollte vor jedem (neuen) aktiven Arbeiten gemacht werden, um nicht auf alten Datenstämmen zu schaffen und Konflikte zu provozieren
  • Merge Conflicts
    • Ich mache eine Änderung (mit commit ohne push?)
    • Gleichzeitig macht jemand anderes Änderungen und committet
    • Wenn ich jetzt pullen will, gibt es ggfs. einen merge conflict
    • im Editor (WebStorm) ist der Head meiner Version mit <<< Zeichen und
    • nach den === Zeichen
    • die Online Version, die mit >>> endet
    • Ich muss nun manuell die beiden Versionen zusammenführen im Webstorm
    • und nochmal einen commit und push machen mit den konsolidierten Änderungen
    • Danach sind alle drei commits auf GitHub sichtbar
  • Log
    • git log: mit q können wir den log dann wieder verlassen
    • git log --oneline: Log in kompakter Form
  • Restore
    • git restore [filename]
    • nach Änderungen, aber vor einem Commit kann die ursprüngliche Version wieder hergestellt werden
    • Änderungen werden rückgängig gemacht)
  • Reset
    • git reset --hard [commit hash]: zu bestimmten Commit zurück
    • git reset --hard origin/master: zur aktuellen Remote Version zurück
  • Branching
    • der "Head" zeigt auf den aktuellen Branch, wo ich arbeite und kann verändert werden
    • git branch: zeigt mir den aktuellen Branch an
    • git checkout -b [branch_name]: neuer Branch wird erstellt
    • git checkout [branche_name]: hier wechselt man den Branch
    • wenn ich vom neuen Branch einen commit machen, muss ein zusätzlicher Befehl ausgeführt werden, sodass ein Merge Pull Request in GutHub erstellt wird. Den kann ich auf GitHub easy bearbeiten und alles zusammenführen.
    • git merge [branch name]: wenn ich auf dem main bin, kann ich einen anderen mergen, aber Konflikte sind möglich
